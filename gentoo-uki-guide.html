How it all fits together overview
▼
🔧
dracut → builds initramfs
↓
🔩
systemd-utils (stub loader) → bundles kernel + initramfs + cmdline
↓
📦
single .efi file → placed in /efi/EFI/Gentoo/
↓
⚡
UEFI boots it directly → no GRUB, no secondary bootloader
ℹ The kernel command line (root UUID etc.) is baked into the .efi file — you don't need to pass it at boot time.
01
Find your UUIDs and disk info required info
▼
⚠ Run these first and note your root partition UUID and EFI partition (the small ~512MB one). You'll need them in Step 5.
bash
blkid
bash
lsblk
ℹ From blkid: note the UUID next to your root partition (e.g. /dev/sda3). From lsblk: note which partition is your EFI (e.g. /dev/sda1).
02
Set USE flags /etc/portage
▼
bash
mkdir -p /etc/portage/package.use echo 'sys-apps/systemd-utils boot kernel-install' >> /etc/portage/package.use/uki echo 'sys-kernel/installkernel dracut uki' >> /etc/portage/package.use/uki
03
Emerge required packages portage
▼
bash
emerge --ask sys-apps/systemd-utils
bash
emerge --ask sys-kernel/installkernel
04
Create EFI directory /efi
▼
bash
mkdir -p /efi/EFI/Gentoo
ℹ EFI stub requires kernels installed to /efi, not /boot. Make sure your EFI partition is mounted at /efi.
05
Create dracut configs ⚠ replace UUID
▼
⚠ Replace YOUR-ROOT-UUID-HERE with your actual UUID from Step 1 in both commands below.
bash
mkdir -p /etc/dracut.conf.d
/etc/dracut.conf.d/00-installkernel.conf — note the leading and trailing spaces inside the quotes
kernel_cmdline=" root=UUID=YOUR-ROOT-UUID-HERE rw "
bash
echo 'kernel_cmdline=" root=UUID=YOUR-ROOT-UUID-HERE rw "' > /etc/dracut.conf.d/00-installkernel.conf
/etc/dracut.conf.d/uki.conf
uefi="yes" kernel_cmdline="root=UUID=YOUR-ROOT-UUID-HERE rw quiet"
bash
printf 'uefi="yes"\nkernel_cmdline="root=UUID=YOUR-ROOT-UUID-HERE rw quiet"\n' > /etc/dracut.conf.d/uki.conf

Verify both files:

bash
cat /etc/dracut.conf.d/00-installkernel.conf cat /etc/dracut.conf.d/uki.conf
06
Install the kernel portage
▼
✓ This will automatically build the kernel, generate an initramfs via dracut, bundle everything into a .efi UKI, and place it in /efi/EFI/Gentoo/.
bash
emerge --ask sys-kernel/gentoo-kernel-bin
ℹ Prefer compiling from source? Use sys-kernel/gentoo-kernel instead.
07
Create UEFI boot entry ⚠ replace disk/part
▼
bash
emerge --ask sys-fs/efibootmgr
⚠ Replace /dev/sda and 1 with your actual disk and EFI partition number from Step 1. Example: if EFI is /dev/nvme0n1p1 then use --disk /dev/nvme0n1 --part 1.
bash
efibootmgr --create \ --disk /dev/sda \ --part 1 \ --label "Gentoo" \ --loader "EFI\Gentoo\gentoo.efi"

Verify the entry was created:

bash
efibootmgr -v
ℹ No kernel cmdline is passed here — it's already baked into the UKI.
08
Set dist-kernel USE flag future upgrades
▼
✓ This ensures external modules (AMD GPU drivers, ZFS, etc.) and the UKI are automatically rebuilt when the kernel upgrades.
bash
echo 'USE="dist-kernel"' >> /etc/portage/make.conf
09
Verify and reboot final
▼

Check the UKI was placed in the EFI directory:

bash
ls /efi/EFI/Gentoo/
✓ You should see a .efi file. If you do, you're ready to reboot!
bash
reboot
