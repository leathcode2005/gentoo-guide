Disk Identification — UUID Is Everything

With two NVMe drives, Linux assigns device names (nvme0n1, nvme1n1) by boot enumeration order — which is not stable across reboots. UUID is burned into the partition and never changes. This guide uses UUID for every config file reference.

🚫

Never use /dev/nvme0n1p2 or any /dev/nvme* path in any config file. They will randomly fail on a dual-NVMe system.

Identify your drives before touching anything

live environment
lsblk -d -o NAME,SIZE,MODEL   # see all drives and their sizes/models
lsblk -f                     # tree view of partitions and filesystems

# Decide which NVMe will be your Gentoo install drive.
# This guide uses nvme0n1 as the example — substitute your actual drive.
# WARNING: partitioning will destroy all existing data on that drive.
⚠️

Back up anything important on the target drive. The next step wipes it.

1

Partition the Disk (GPT + UEFI)

Three partitions: EFI System Partition (1GB), swap (match your RAM), root (rest of disk). GPT disklabel is required for UEFI boot.

📌

The handbook recommends 1GB EFI and at least 4GB swap. On a modern Ryzen system, matching swap to RAM (16GB, 32GB, 64GB) is sensible.

Open fdisk on your install drive

live environment — replace nvme0n1 with your drive
fdisk /dev/nvme0n1

fdisk commands — type these in order

inside fdisk
g          # create new GPT disklabel — WIPES ALL EXISTING PARTITIONS

# Partition 1: EFI (1GB)
n          # new partition
           # number: 1 (Enter)
           # first sector (Enter)
+1G        # size
t          # change type
1          # EFI System

# Partition 2: swap (adjust +32G to match your RAM)
n
           # number: 2 (Enter)
           # first sector (Enter)
+32G       # match your RAM: +16G, +32G, +64G etc.
t
2          # select partition 2
19         # Linux swap

# Partition 3: root (rest of disk)
n
           # number: 3 (Enter)
           # first sector (Enter)
           # last sector (Enter — uses all remaining space)
t
3          # select partition 3
23         # Linux root (x86-64)

p          # print table to verify before saving
w          # write to disk — POINT OF NO RETURN

Expected layout

what 'p' should show
Device           Size  Type
/dev/nvme0n1p1    1G   EFI System
/dev/nvme0n1p2   32G   Linux swap
/dev/nvme0n1p3  <rest> Linux root (x86-64)
2

Format Partitions & Mount

Format each partition and mount them. Device paths here are fine — these are one-time commands, not config files.

⚠️

mkfs erases all data on the target partition. Verify you're targeting the right drive.

Format EFI (must be FAT32)

live environment
mkfs.fat -F 32 /dev/nvme0n1p1

Format and activate swap

live environment
mkswap /dev/nvme0n1p2
swapon /dev/nvme0n1p2

Format root (ext4)

live environment
mkfs.ext4 /dev/nvme0n1p3

Mount partitions

live environment
mount /dev/nvme0n1p3 /mnt/gentoo
mkdir -p /mnt/gentoo/efi
mount /dev/nvme0n1p1 /mnt/gentoo/efi
💡

Run blkid now and write down the UUID for each partition — they're set during formatting and won't change. You'll need all three UUIDs in later steps.

live — write these down now
blkid
# nvme0n1p1  short UUID like  A1B2-C3D4                            ← EFI
# nvme0n1p2  long UUID like   xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx  ← swap
# nvme0n1p3  long UUID like   2122cd72-94d7-4dcc-821e-3705926deecc  ← root
3

Download & Unpack Stage File

Use the plain OpenRC stage3 for amd64 — not the desktop variant. The plain stage is for CLI/server systems and is exactly right for a minimal installation.

📌

There are two OpenRC stage3 options: stage3-amd64-openrc (225MB, plain CLI) and stage3-amd64-desktop-openrc (603MB, includes llvm/rust/desktop libs). For this minimal system use the plain openrc — the one without "desktop" in the name. The profile selected later must also match: default/linux/amd64/23.0, not a desktop sub-profile.

Sync the clock

live environment
date         # HTTPS downloads require accurate time
chronyd -q   # auto-sync if off by more than a few minutes

Download

live environment
cd /mnt/gentoo
links https://www.gentoo.org/downloads/mirrors/
# Navigate: releases → amd64 → autobuilds → current-stage3-amd64-openrc/
# Make sure it says "openrc" NOT "desktop-openrc"
# Press 'd' on the stage3-amd64-openrc-[DATE].tar.xz file to download

Unpack

live environment
tar xpvf stage3-*.tar.xz --xattrs-include='*.*' --numeric-owner -C /mnt/gentoo
📌

p preserves permissions, --xattrs-include='*.*' preserves extended attributes, --numeric-owner preserves UID/GID exactly as Gentoo's release team intended.

4

Enter the Chroot

From this step on, all commands run inside your new Gentoo system. If the install is interrupted, you can resume by re-mounting and re-chrooting — no need to re-partition.

Copy DNS and bind virtual filesystems

live environment
cp --dereference /etc/resolv.conf /mnt/gentoo/etc/

mount --types proc /proc /mnt/gentoo/proc
mount --rbind /sys /mnt/gentoo/sys
mount --make-rslave /mnt/gentoo/sys
mount --rbind /dev /mnt/gentoo/dev
mount --make-rslave /mnt/gentoo/dev
mount --bind /run /mnt/gentoo/run
mount --make-slave /mnt/gentoo/run

Chroot in

enter chroot
chroot /mnt/gentoo /bin/bash
source /etc/profile
export PS1="(chroot) ${PS1}"
📌

Your prompt should now show (chroot). All remaining steps until the reboot are run inside this chroot. If the install is ever interrupted, re-mount your partitions and re-run these three lines to resume.

Mount EFI inside the chroot

chroot — replace YOUR-EFI-UUID with your EFI partition UUID
mkdir -p /efi/EFI/Gentoo
mount UUID=YOUR-EFI-UUID /efi

Configure make.conf

📌

Use nano here — vim is not installed yet. Portage itself isn't synced until the next step so you cannot emerge anything yet. Vim gets installed in Step 6 immediately after emerge-webrsync.

chroot
nano /etc/portage/make.conf
/etc/portage/make.conf — full contents
# Compiler flags — native targets your exact AMD CPU
COMMON_FLAGS="-march=native -O2 -pipe"
CFLAGS="${COMMON_FLAGS}"
CXXFLAGS="${COMMON_FLAGS}"

# Parallel jobs — run nproc to check your thread count
# Rule: at least 2GB RAM per job (e.g. -j16 needs 32GB RAM)
MAKEOPTS="-j16 -l17"

# USE flags
USE="dist-kernel networkmanager"

# AMD GPU drivers
VIDEO_CARDS="amdgpu radeonsi"

# Accept pre-compiled binary firmware blobs (nothing compiled from source)
ACCEPT_LICENSE="-* @FREE @BINARY-REDISTRIBUTABLE"

Firmware & Microcode

AMD CPU microcode is bundled inside linux-firmware — no separate microcode package needed. SOF firmware is required for Ryzen integrated audio.

📌

@BINARY-REDISTRIBUTABLE permits installing pre-compiled binary blobs. Nothing is compiled — Portage copies .bin/.fw files to /lib/firmware/. The license gate is purely due to proprietary redistribution terms.

Accept firmware license (only if emerge rejects it)

chroot
echo 'sys-kernel/linux-firmware @BINARY-REDISTRIBUTABLE' >> /etc/portage/package.license

Install firmware packages

chroot
emerge --ask sys-kernel/linux-firmware    # AMD GPU, WiFi, CPU microcode
emerge --ask sys-firmware/sof-firmware     # Ryzen integrated audio firmware

Configure /etc/fstab

All three partitions use UUID. Never use /dev/nvme* paths here.

⚠️

Replace YOUR-EFI-UUID, YOUR-SWAP-UUID, and YOUR-ROOT-UUID with your actual values from blkid before running.

chroot — replace all 3 UUIDs first
cat > /etc/fstab <<'EOF'
UUID=YOUR-EFI-UUID    /efi   vfat   umask=0077,tz=UTC   0 2
UUID=YOUR-SWAP-UUID   none   swap   sw                  0 0
UUID=YOUR-ROOT-UUID   /      ext4   defaults,noatime    0 1
EOF

Configure installkernel for UKI

installkernel drives the whole kernel install pipeline: dracut builds the initramfs, systemd-stub bundles it into a UKI, and the .efi lands in /efi/EFI/Gentoo/.

🚫

Do NOT add systemd or systemd-boot USE flags. sys-apps/systemd-utils provides only the EFI stub loader — it does not touch OpenRC.

chroot
echo 'sys-apps/systemd-utils boot kernel-install' >> /etc/portage/package.use/installkernel
echo 'sys-kernel/installkernel dracut uki' >> /etc/portage/package.use/installkernel

emerge --ask sys-apps/systemd-utils
emerge --ask sys-kernel/installkernel
14

Dracut Config — Embed Root UUID

The root UUID must be embedded in the kernel command line so the system can find root at boot — there's no bootloader in the chain to supply it. Three files are required. installkernel's chroot detection script checks each one in turn and fails if none are found.

⚠️

Replace YOUR-ROOT-UUID in all commands below with your actual root partition UUID from blkid. Run blkid | grep ext4 if you need to look it up.

🚫

All three files are required — do not skip any. installkernel's chroot check (05-check-chroot.install) searches for a cmdline across multiple locations. If none exist the kernel post-install fails, even if your prompt says (chroot). Make sure you are inside the chroot before running these.

Create all three cmdline files

chroot — replace YOUR-ROOT-UUID in every command
mkdir -p /etc/dracut.conf.d /etc/cmdline.d /etc/kernel

# File 1: checked by installkernel's chroot detection script
echo "root=UUID=YOUR-ROOT-UUID quiet" > /etc/kernel/cmdline

# File 2: used by installkernel's dracut integration
echo 'kernel_cmdline=" root=UUID=YOUR-ROOT-UUID quiet "' > /etc/dracut.conf.d/00-installkernel.conf
#                              ^ space required       ^ space required — keep both

# File 3: used by dracut when building the UKI
printf 'uefi="yes"\nkernel_cmdline="root=UUID=YOUR-ROOT-UUID quiet"\n' > /etc/dracut.conf.d/uki.conf

Verify all files were written correctly

Run these before proceeding to the kernel install. If any of them show nothing or the wrong UUID, fix it now — not after a failed emerge.

chroot — all four should show your UUID
cat /etc/kernel/cmdline
cat /etc/dracut.conf.d/00-installkernel.conf
cat /etc/dracut.conf.d/uki.conf
grep 'root=UUID' /etc/kernel/cmdline /etc/dracut.conf.d/*.conf   # all three in one shot

Verify EFI is mounted and has space

chroot
mount | grep /efi   # must show your EFI partition mounted
df -h /efi          # need 100MB+ free
ls /efi/EFI/Gentoo  # directory must exist
15

Install the Kernel

chroot
emerge --ask sys-kernel/gentoo-kernel

Confirm the UKI was created

chroot
ls -lh /efi/EFI/Gentoo/    # should show .efi file(s)
strings /efi/EFI/Gentoo/gentoo-*.efi | grep 'root=UUID'    # confirm UUID embedded
💡

If emerge fails with "Dracut will be run from inside a chroot but no cmdline configured": run cat /etc/kernel/cmdline and cat /etc/dracut.conf.d/00-installkernel.conf. If either file is missing or empty, the commands in Step 14 were not run inside the chroot. Re-enter the chroot, recreate the files, then retry with emerge --config '=sys-kernel/gentoo-kernel-6.18.18:6.18.18'.

16

Register the UEFI Boot Entry

One-time operation. Writes a boot entry into UEFI NVRAM pointing to your UKI.

chroot — replace gentoo-x.y.z.efi with your actual filename
emerge --ask sys-boot/efibootmgr

ls /efi/EFI/Gentoo/    # get exact .efi filename first

efibootmgr --create --disk /dev/nvme0n1 --part 1 \
  --label "Gentoo Linux" \
  --loader "\EFI\Gentoo\gentoo-x.y.z.efi"

efibootmgr -v    # verify entry was added
📌

The backslash in --loader is required by UEFI spec. Using /dev/nvme0n1 for --disk here is fine — it's a one-time command to write NVRAM, not a config file read at boot.
