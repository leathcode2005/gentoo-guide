Gentoo Installation Guide (OpenRC + UEFI + ext4)

This guide is intentionally minimal. It keeps only required steps for a standard OpenRC install on UEFI hardware.

1) Prepare Disk

Identify target disk:


lsblk -o NAME,SIZE,TYPE,FSTYPE,MOUNTPOINT

Start partitioning (replace `/dev/sda` as needed):


fdisk /dev/sda

Inside `fdisk`, create GPT + 3 partitions:


g
n
1
<Enter>
+1G
t
1
n
2
<Enter>
+8G
t
2
19
n
3
<Enter>
<Enter>
w

Note on partition types: After creating partition 1, t automatically selects it (the only partition at that point) and 1 sets the type code to EFI System (GPT type 1). Without this, the partition would default to "Linux filesystem" and firmware would not recognize it as an ESP. For partition 2, t → 2 → 19 sets the swap type.

Resulting layout:

  • `/dev/sda1`: EFI System Partition
  • `/dev/sda2`: swap
  • `/dev/sda3`: root

2) Create Filesystems and Mount

Format partitions:


mkfs.vfat -F 32 /dev/sda1
mkswap /dev/sda2
swapon /dev/sda2
mkfs.ext4 /dev/sda3

Mount target root and EFI:


mkdir -p /mnt/gentoo
mount /dev/sda3 /mnt/gentoo
mkdir -p /mnt/gentoo/efi
mount /dev/sda1 /mnt/gentoo/efi

3) Download and Extract Stage3

Go to target root:


cd /mnt/gentoo

Sync time for HTTPS:


chronyd -q

Download OpenRC stage3 (use current filename from Gentoo mirror):


wget https://distfiles.gentoo.org/releases/amd64/autobuilds/current-stage3-amd64-openrc/stage3-amd64-openrc-<date>.tar.xz

Extract stage3 (run from within /mnt/gentoo):


tar xpvf stage3-*.tar.xz --xattrs-include='*.*' --numeric-owner

4) Configure Portage (make.conf)

Set compiler defaults before entering the chroot so build settings are in effect for the first Portage sync:


cat > /mnt/gentoo/etc/portage/make.conf <<'EOF'
COMMON_FLAGS="-march=native -O2 -pipe"
CFLAGS="${COMMON_FLAGS}"
CXXFLAGS="${COMMON_FLAGS}"
MAKEOPTS="-j$(nproc)"
EOF

5) Chroot

Copy DNS config:


cp --dereference /etc/resolv.conf /mnt/gentoo/etc/

Enter the chroot using arch-chroot (preferred — handles pseudo-filesystem mounts automatically):


arch-chroot /mnt/gentoo
source /etc/profile
export PS1="(chroot) ${PS1}"

Note: arch-chroot is available on Gentoo's official install media and handles pseudo-filesystem mounts automatically. If it is not available on your live environment, use the manual method below instead.

Manual fallback — mount pseudo-filesystems and enter chroot by hand:


mount --types proc /proc /mnt/gentoo/proc
mount --rbind /sys /mnt/gentoo/sys
mount --make-rslave /mnt/gentoo/sys
mount --rbind /dev /mnt/gentoo/dev
mount --make-rslave /mnt/gentoo/dev
mount --bind /run /mnt/gentoo/run
mount --make-slave /mnt/gentoo/run
chroot /mnt/gentoo /bin/bash
source /etc/profile
export PS1="(chroot) ${PS1}"

Verify network connectivity inside the chroot:


ping -c 3 gentoo.org

6) Portage and Profile

Install repository snapshot:


emerge-webrsync

Set profile:


eselect profile list
eselect profile set <openrc-profile-number>

Update the system to conform to the new profile's USE flags (recommended by the official Handbook):


emerge --ask --verbose --update --deep --newuse @world

6a) CPU Flags

Install cpuid2cpuflags to detect CPU-specific USE flags (AES-NI, AVX2, SSE4, etc.) and apply them so packages are compiled with optimal CPU optimizations:


emerge --ask app-portage/cpuid2cpuflags
echo "*/* $(cpuid2cpuflags)" > /etc/portage/package.use/00cpu-flags

7) Basic System Settings

Set timezone to Detroit (OpenRC method):


echo "America/Detroit" > /etc/timezone
emerge --config sys-libs/timezone-data

Note: Replace America/Detroit with your own timezone. List available timezones with ls /usr/share/zoneinfo/.

Set locale:


echo 'en_US.UTF-8 UTF-8' > /etc/locale.gen
locale-gen
eselect locale list
eselect locale set <en_US-utf8-number>
env-update && source /etc/profile

8) Kernel

Configure installkernel to use dracut and efistub:


mkdir -p /etc/portage/package.use
cat > /etc/portage/package.use/installkernel <<'EOF'
sys-kernel/installkernel dracut efistub
EOF

Note: The efistub flag tells installkernel to automatically copy the kernel and initramfs to the EFI System Partition on every kernel install, so future kernel updates require no manual file copying.

Set root UUID for initramfs:


ROOT_UUID=$(blkid -s UUID -o value /dev/sda3)
mkdir -p /etc/dracut.conf.d
cat > /etc/dracut.conf.d/00-installkernel.conf <<EOF
kernel_cmdline=" root=UUID=${ROOT_UUID} rw "
EOF

Install firmware, kernel tooling, and kernel:


emerge --ask sys-kernel/linux-firmware sys-kernel/installkernel sys-kernel/gentoo-kernel

9) Configure System Files

Set hostname:


echo gentoo > /etc/hostname

Create fstab:


ESP_UUID=$(blkid -s PARTUUID -o value /dev/sda1)
SWAP_UUID=$(blkid -s PARTUUID -o value /dev/sda2)
ROOT_PARTUUID=$(blkid -s PARTUUID -o value /dev/sda3)
cat > /etc/fstab <<EOF
PARTUUID=${ESP_UUID}      /efi   vfat  umask=0077,tz=UTC  0 2
PARTUUID=${SWAP_UUID}     none   swap  sw                  0 0
PARTUUID=${ROOT_PARTUUID} /      ext4  defaults,noatime    0 1
EOF

Set hosts file:


cat > /etc/hosts <<'EOF'
127.0.0.1   gentoo.localdomain gentoo localhost
::1         gentoo.localdomain gentoo localhost
EOF

Set root password:


passwd

10) Networking and Required Services

Install and enable DHCP client:


emerge --ask net-misc/dhcpcd
rc-update add dhcpcd default

Install and enable logger + cron + time sync:


emerge --ask app-admin/sysklogd sys-process/cronie net-misc/chrony
rc-update add sysklogd default
rc-update add cronie default
rc-update add chronyd default

11) UEFI Boot (EFI Stub, No GRUB)

Install efibootmgr:


emerge --ask sys-boot/efibootmgr

Note: installkernel with the efistub USE flag (configured in Step 8) already placed the kernel and initramfs on the ESP under /efi/EFI/Gentoo/ during the emerge in Step 8. No manual copying is needed.

Check the exact filenames installkernel placed on the ESP:


ls /efi/EFI/Gentoo/

Re-derive the root UUID (in case the shell variable from Step 8 is no longer in scope) and create the persistent firmware boot entry. Replace <vmlinuz-file> and <initramfs-file> with the exact filenames shown by ls above:


ROOT_UUID=$(blkid -s UUID -o value /dev/sda3)
[ -z "${ROOT_UUID}" ] && echo "ERROR: could not read UUID for /dev/sda3 — check blkid output" && exit 1
efibootmgr --create --disk /dev/sda --part 1 --label "Gentoo" --loader "\\EFI\\Gentoo\\<vmlinuz-file>" --unicode "initrd=\\EFI\\Gentoo\\<initramfs-file> root=UUID=${ROOT_UUID} rw"

Note on UUID vs PARTUUID: The fstab (Step 9) uses PARTUUID (partition-level identifier) while the kernel cmdline uses UUID (filesystem-level identifier). Both are valid and work independently; the kernel cmdline UUID is used by the initramfs to find the root filesystem before fstab is processed.

Kernel Updates

Future kernel updates are fully automatic. Just run:


emerge --ask --verbose --update --deep --newuse @world

installkernel + dracut + efistub will rebuild the initramfs and copy the new kernel and initramfs to the ESP automatically. No manual steps are needed after the initial boot entry is created above.

11a) Video / GPU Drivers

Detect the installed GPU:


lspci | grep -i vga

Set VIDEO_CARDS in /etc/portage/make.conf and install the matching driver. Only do this section if you intend to set up a graphical environment (X11/Wayland).

Intel integrated graphics

For modern Intel (Gen 8+ / Broadwell+) the in-tree modesetting driver is preferred:


echo 'VIDEO_CARDS="intel i965"' >> /etc/portage/make.conf
emerge --ask x11-drivers/xf86-video-intel

For very recent Intel (Iris Xe / Gen 12+) use VIDEO_CARDS="intel iris" instead.

AMD / Radeon (open-source amdgpu)


echo 'VIDEO_CARDS="amdgpu radeonsi"' >> /etc/portage/make.conf
# sys-kernel/linux-firmware (installed in Step 8) already includes AMD GPU firmware
emerge --ask x11-drivers/xf86-video-amdgpu

NVIDIA (proprietary)


echo 'VIDEO_CARDS="nvidia"' >> /etc/portage/make.conf
echo "x11-drivers/nvidia-drivers NVIDIA-r2" >> /etc/portage/package.license
emerge --ask x11-drivers/nvidia-drivers

12) Finalize and Reboot

Create daily user:


useradd -m -G users,wheel,audio,video,usb -s /bin/bash larry
passwd larry

Note: Replace larry with your preferred username.

Install sudo and grant the wheel group sudo access:


emerge --ask app-admin/sudo
echo "%wheel ALL=(ALL:ALL) ALL" > /etc/sudoers.d/wheel

Exit, unmount, reboot:


exit
cd /
umount -l /mnt/gentoo/dev{/shm,/pts,} 2>/dev/null || true
umount -R /mnt/gentoo
reboot
