BEFORE YOU START

Find Your Values First

Run these discovery commands first. You'll need the output to fill in the copy-paste commands below. Open a second terminal or write them down.

Find your EFI partition, root UUID, and kernel version:

discovery.sh 
# # 1. List all block devices — find your EFI and root partitions
lsblk -o NAME,SIZE,TYPE,FSTYPE,MOUNTPOINT,UUID

# # 2. Get UUID for your root partition specifically (replace sdaX)
blkid /dev/sdaX

# # 3. Check EFI partition number (usually partition 1 or 2)
fdisk -l | grep -i efi

Write down these values — you'll substitute them below:

VariableHow to find it
ROOT_UUID From blkid output — the UUID of your / partition
EFI_DISK Parent disk of EFI partition, e.g. /dev/sda or /dev/nvme0n1
EFI_PART_NUM Partition number of EFI partition — usually 1 or 2
KEYMAP Your keyboard layout, e.g. us, de, uk
STEP 01

Portage USE Flags & License

Set the dracut USE flag on installkernel and accept the firmware license. These must be done before emerging the kernel.

portage-flags.sh 
# # Enable dracut integration for installkernel
echo "sys-kernel/installkernel dracut" > /etc/portage/package.use/installkernel

# # Accept linux-firmware license (required for wireless/GPU firmware)
echo "sys-kernel/linux-firmware linux-fw-redistributable no-source-code" >> /etc/portage/package.license

# # Verify
cat /etc/portage/package.use/installkernel
cat /etc/portage/package.license
STEP 02

Emerge Kernel Sources & Tools

Install the kernel sources, dracut, zstd compression, efibootmgr, and linux-firmware in one go.

emerge-kernel.sh 
emerge --verbose \
    sys-kernel/gentoo-sources \
    sys-kernel/dracut \
    app-arch/zstd \
    sys-boot/efibootmgr \
    sys-kernel/linux-firmware
Note: This will take a while. linux-firmware is large but needed for most hardware (NIC, GPU, WiFi, NVMe). If you want a minimal install you can skip it and add later.
STEP 03

Configure the Kernel

Symlink the source and open menuconfig. The localmodconfig option pre-selects only drivers for hardware currently detected — highly recommended for a first build.

configure-kernel.sh 
# # Confirm the symlink exists
ls -la /usr/src/linux

# # Change into kernel source directory
cd /usr/src/linux

# # RECOMMENDED: seed config from currently loaded modules (much faster first build)
make localmodconfig

# # Then open the full config menu to review / add anything missing
make menuconfig
Critical options to verify in menuconfig: EFI stub support (CONFIG_EFI_STUB=y), your filesystem driver (ext4/btrfs/xfs — built-in, not module), and your disk controller driver (AHCI/NVMe — built-in).

Quick one-liner to confirm EFI stub is enabled after saving config:

verify-config.sh 
grep -E "CONFIG_EFI_STUB|CONFIG_EFI=|CONFIG_FB_EFI" /usr/src/linux/.config
STEP 04

Build & Install the Kernel

Build using all available CPU threads, then install modules and the kernel image. Replace $(nproc) if you want to limit CPU usage.

build-kernel.sh 
cd /usr/src/linux

# # Build kernel + modules (uses all CPU cores)
make -j$(nproc)

# # Install kernel modules to /lib/modules/
make modules_install

# # Copy kernel image to /boot
make install

# # Confirm kernel is in /boot
ls -lh /boot/
Expected output: You should see a vmlinuz-X.XX.X-gentoo file in /boot. The exact name is used in the next step.
STEP 05

Generate initramfs with dracut

Dracut reads the kernel version from the /usr/src/linux symlink automatically. Run this block as-is — it will detect the version for you.

dracut-initramfs.sh 
# # Read kver from the /usr/src/linux symlink
kver="$(readlink /usr/src/linux)"
kver="${kver#linux-}"
echo "Kernel version: $kver"

# # Mount EFI partition if not already mounted
mount /boot/efi 2>/dev/null || echo "already mounted or not in fstab yet — mount manually if needed"

# # Copy kernel to EFI partition
kernel_file="$(find /boot \( -name 'vmlinuz-*' -or -name 'kernel-*' \) -printf '%f\n' | sort -V | tail -n 1)"
echo "Using kernel: $kernel_file"
cp "/boot/$kernel_file" /boot/efi/vmlinuz.efi

# # Generate initramfs — output directly to EFI partition
dracut \
    --kver        "$kver" \
    --zstd \
    --no-hostonly \
    --ro-mnt \
    --add         "bash" \
    --force \
    /boot/efi/initramfs.img

# # Confirm both files exist on EFI partition
ls -lh /boot/efi/
Using LUKS, RAID, Btrfs, or ZFS? Add the relevant module to the --add flag:

LUKS: --add "bash crypt"  |   RAID: --add "bash mdraid"  |   Btrfs: --add "bash btrfs"  |   ZFS: --add "bash zfs"
STEP 06

Create the EFI Boot Entry

Substitute your values here. Replace YOUR_ROOT_UUID, /dev/sda (EFI disk), 1 (EFI partition number), and us (keymap) before running.
efibootmgr.sh — EDIT BEFORE RUNNING 
# # ── EDIT THESE FOUR VALUES ────────────────────────────────────────
ROOT_UUID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"   # from blkid
EFI_DISK="/dev/sda"                               # parent disk (not partition)
EFI_PART_NUM="1"                                  # EFI partition number
KEYMAP="us"                                       # keyboard layout
# ─────────────────────────────────────────────────────────────────

efibootmgr \
    --verbose \
    --create \
    --disk    "$EFI_DISK" \
    --part    "$EFI_PART_NUM" \
    --label   "gentoo" \
    --loader  '\vmlinuz.efi' \
    --unicode "initrd=\initramfs.img rd.vconsole.keymap=$KEYMAP root=UUID=$ROOT_UUID"

Verify the entry was created and check boot order:

verify-efi.sh 
# # List all EFI boot entries
efibootmgr --verbose

# # Confirm kernel + initramfs are present on EFI partition
ls -lh /boot/efi/
NVMe drives: Your disk will be /dev/nvme0n1 and the partition /dev/nvme0n1p1. Pass --disk /dev/nvme0n1 --part 1 to efibootmgr.
STEP 07

Generate fstab

Add entries for root, EFI, and optionally swap. Always use UUIDs — never device names like /dev/sda1.

fstab.sh — EDIT BEFORE RUNNING 
# # ── EDIT THESE VALUES ─────────────────────────────────────────────
ROOT_UUID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"   # UUID of / partition
EFI_UUID="XXXX-XXXX"                              # UUID of EFI partition (short format)
ROOT_FS="ext4"                                    # ext4 / btrfs / xfs
# ─────────────────────────────────────────────────────────────────

# # Back up existing fstab
cp /etc/fstab /etc/fstab.bak

# # Write root entry
printf '%-46s  %-16s  %-6s  %-40s %s\n' \
    "UUID=$ROOT_UUID" "/" "$ROOT_FS" "defaults,noatime" "0 1" \
    >> /etc/fstab

# # Write EFI entry
printf '%-46s  %-16s  %-6s  %-40s %s\n' \
    "UUID=$EFI_UUID" "/boot/efi" "vfat" \
    "defaults,noatime,fmask=0177,dmask=0077,noexec,nodev,nosuid" "0 2" \
    >> /etc/fstab

# # Review the result
cat /etc/fstab

If you have a swap partition, add it too:

fstab-swap.sh — optional 
SWAP_UUID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"   # UUID of swap partition

printf '%-46s  %-16s  %-6s  %-40s %s\n' \
    "UUID=$SWAP_UUID" "none" "swap" "defaults,discard" "0 0" \
    >> /etc/fstab
STEP 08

Final Checks & Reboot

Run this checklist before rebooting out of the chroot.

preflight-check.sh 
# # 1. Kernel image on EFI partition
ls -lh /boot/efi/vmlinuz.efi

# # 2. Initramfs on EFI partition
ls -lh /boot/efi/initramfs.img

# # 3. EFI boot entry exists
efibootmgr | grep gentoo

# # 4. fstab looks sane
cat /etc/fstab

# # 5. Network service is enabled (openrc)
rc-update show | grep -E "dhcpcd|networkmanager|net\."

# # 6. Set root password if not done
passwd root
exit-chroot-and-reboot.sh 
# # Exit chroot
exit

# # Back on live environment — unmount everything
umount -l /mnt/gentoo/dev{/shm,/pts,}
umount -R /mnt/gentoo

# # Reboot
reboot
If boot fails: Boot back into the live USB, mount your root partition to /mnt/gentoo, chroot back in with ./install --chroot /mnt/gentoo or manually, and re-check the efibootmgr entry and dracut initramfs.

✓ Setup Complete

If all preflight checks passed, your system should boot into Gentoo.
First boot may take slightly longer as services initialize for the first time.
