Installation Steps
STEP 01
Find your disk and partition info // required
▼
⚠ Run these first. Note your root partition UUID and your EFI partition (the small ~512MB partition, usually /dev/sda1 or /dev/nvme0n1p1).
bash
blkid
bash
lsblk
ℹ Example output from blkid: /dev/sda3: UUID="2122cd72-94d7-4dcc-821e-3705926deecc" TYPE="ext4" — the long string after UUID= is what you need.
STEP 02
Install linux-firmware and microcode // AMD
▼
✓ AMD microcode is bundled inside linux-firmware — no separate package needed unlike Intel.
bash
emerge --ask sys-kernel/linux-firmware
ℹ If prompted about licenses, accept them. Most wireless cards and GPUs also need firmware from this package.
STEP 03
Set USE flags for installkernel // grub + dracut
▼
/etc/portage/package.use/installkernel
sys-kernel/installkernel grub dracut
bash
mkdir -p /etc/portage/package.use echo 'sys-kernel/installkernel grub dracut' > /etc/portage/package.use/installkernel
ℹ The grub flag makes installkernel automatically run grub-mkconfig after each kernel install. The dracut flag auto-generates an initramfs.
STEP 04
Emerge installkernel // portage
▼
bash
emerge --ask sys-kernel/installkernel
STEP 05
Configure dracut with root UUID // ⚠ replace UUID
▼
⚠ Replace YOUR-ROOT-UUID-HERE with your actual root partition UUID from Step 1. This is required — installkernel will fail in a chroot without it.
bash
mkdir -p /etc/dracut.conf.d
/etc/dracut.conf.d/00-installkernel.conf — note the leading and trailing spaces inside the quotes
kernel_cmdline=" root=UUID=YOUR-ROOT-UUID-HERE rw "
bash
echo 'kernel_cmdline=" root=UUID=YOUR-ROOT-UUID-HERE rw "' > /etc/dracut.conf.d/00-installkernel.conf

Verify it looks correct:

bash
cat /etc/dracut.conf.d/00-installkernel.conf
STEP 06
Install the kernel // dist-kernel
▼
✓ This installs the kernel to /boot, generates an initramfs, and calls installkernel automatically. Future upgrades are handled by the package manager.
bash — precompiled (fastest)
emerge --ask sys-kernel/gentoo-kernel-bin
ℹ Prefer compiling from source? Use sys-kernel/gentoo-kernel instead. Takes longer but allows custom config.
bash — compiled from source (optional)
emerge --ask sys-kernel/gentoo-kernel
STEP 07
Install and configure GRUB // ⚠ replace partition
▼
bash
emerge --ask sys-boot/grub
⚠ Make sure your EFI partition is mounted at /efi or /boot/efi before running the next command. Check with mount | grep efi.
bash — mount EFI partition if not already mounted
mount | grep efi
⚠ Replace /dev/sda below with your actual disk (not partition — no number at the end). Check with lsblk from Step 1. For NVMe it would be /dev/nvme0n1.
bash — install GRUB to EFI
grub-install --target=x86_64-efi --efi-directory=/efi --bootloader-id=Gentoo
ℹ If your EFI partition is mounted at /boot/efi instead, change --efi-directory=/efi to --efi-directory=/boot/efi.

Then generate the GRUB config:

bash
grub-mkconfig -o /boot/grub/grub.cfg
✓ You should see output mentioning Found linux image and Found initrd image. If so, GRUB has detected your kernel successfully.
STEP 08
Set dist-kernel USE flag // future upgrades
▼
✓ This ensures external kernel modules (AMD GPU drivers, ZFS, etc.) and the initramfs are automatically rebuilt on kernel upgrades. GRUB config is also regenerated automatically.
bash
echo 'USE="dist-kernel"' >> /etc/portage/make.conf
STEP 09
Verify and reboot // final check
▼

Confirm kernel and initramfs are in /boot:

bash
ls /boot/

Confirm GRUB config was generated:

bash
cat /boot/grub/grub.cfg | grep menuentry
✓ You should see at least one menuentry line. You're ready to reboot into your new Gentoo system.
bash
reboot
ℹ At the GRUB menu you can press E to edit kernel parameters on the fly — handy for troubleshooting without rebuilding anything.
